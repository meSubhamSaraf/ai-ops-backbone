# AI Ops Backbone

AI-powered operational intelligence system.

Slack → Raw Storage → Gemini Extraction → Structured Action Items

---

## 🚀 What This Project Does

- Ingests Slack messages via webhook
- Stores raw messages in PostgreSQL (Supabase)
- Extracts action items using Gemini
- Normalizes natural-language dates
- Resolves client identity from Slack channel mapping
- Stores structured action items
- Prevents duplicate webhook processing (idempotent design)

---

## 🏗 Architecture Overview

Slack  
→ `/slack/events`  
→ raw_messages (unstructured)  
→ Gemini extraction  
→ action_items (structured)  

Client identity is resolved using `client_channels` table.

---

# 🗄 Database Setup

Run the following SQL commands in Supabase SQL Editor (or PostgreSQL).

---

## Create `raw_messages`

```sql
CREATE TABLE raw_messages (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    source text,
    external_id text,
    content text,
    metadata jsonb,
    processed boolean DEFAULT false,
    created_at timestamptz DEFAULT now()
);
```

---

## Prevent Duplicate Slack Events

```sql
ALTER TABLE raw_messages
ADD CONSTRAINT unique_external_id UNIQUE (external_id);
```

---

## Create `action_items`

```sql
CREATE TABLE action_items (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    client text,
    description text,
    owner text,
    due_date date,
    priority text,
    source_message_id uuid REFERENCES raw_messages(id),
    created_at timestamptz DEFAULT now()
);
```

---

## Create `client_channels`

```sql
CREATE TABLE client_channels (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    client_name text NOT NULL,
    platform text NOT NULL,
    external_id text NOT NULL,
    external_name text,
    is_active boolean DEFAULT true,
    created_at timestamptz DEFAULT now(),
    UNIQUE(platform, external_id)
);
```

---

## Example Client Mapping

```sql
INSERT INTO client_channels (client_name, platform, external_id, external_name)
VALUES ('Leisure Loyalty', 'slack', 'C0838473', 'zh-leisure-loyalty');
```

Replace `C0838473` with your Slack channel ID.

# 🔌 Slack Setup

1. Create Slack App
2. Enable Events API
3. Add Bot Scopes:
   - channels:history
   - channels:read
4. Subscribe to Bot Event:
   - message.channels
5. Set Request URL:

```
https://your-domain/slack/events
```

For local testing:

```bash
ngrok http 8000
```

Use the HTTPS URL generated by ngrok.

---

# 🧠 Processing Flow

1. Slack message arrives
2. Stored in `raw_messages`
3. Client resolved via `client_channels`
4. Gemini extracts action items
5. Dates normalized using python-dateutil
6. `action_items` inserted
7. Message marked processed
8. Duplicate Slack retries ignored
